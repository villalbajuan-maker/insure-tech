# Roadmap

## Phase 1: Product Foundation

- Define glossary and insurance concepts.
- Model the minimum entities for policy and risk analysis.
- Create example scenarios and expected gap findings.
- Define the structure of the analysis report.

## Phase 2: Rules Engine Prototype

- Implement policy and exposure schemas.
- Encode first-pass gap detection rules.
- Run the engine against sample policy scenarios.
- Validate that recommendations are explainable.

## Phase 3: Input Normalization

- Introduce structured ingestion for policy data.
- Normalize coverage terms into a consistent internal format.
- Map raw policy inputs to the domain model.

## Phase 4: Decision Support Experience

- Build a simple user workflow for submitting policy data.
- Show findings, severity, rationale, and recommended next steps.
- Support comparison between current and target coverage.

## Phase 5: Expansion

- Add more coverage lines and industry-specific templates.
- Support benchmarking by business profile.
- Explore document intelligence and assisted extraction.
